Scaling & spacing System

All calculations are based on a 1440px static design, a common Figma viewport size.
In just one minute, the scaling system can be updated in the SCSS file to perfectly match the sizing system of your design. 

            $screen: 1440;
            $sizes: 4, 8, 12, 20, 32, 52, 84, 136, 220, 356, 576, 712;
            $name-by-size: true; // Uses actual size values as variable names

The more _ (underscore) you use, the stronger the resizing effect will be. This is only visible on tablet and mobile viewport sizes.

Use a single - (dash) to minimize the resize effect.

Suffixes like px and rem are here to remind us of the good old days of CSS.

With $name-by-size: true, CSS variables are named using the actual pixel values from your design:

$name-by-size: false
Uses sequential indexes:
--1, --2, --3, --4, --5, --6...
$name-by-size: true (default)
Uses actual pixel values:
--4, --8, --12, --20, --32, --52...

This makes it easier to understand which size you're using directly from your Figma design!

Fluid properties

I use Emmet CSS naming conventions, employing intuitive prefixes for efficient and consistent styling.

w == width

h == height

p == padding

px == padding-inline

py == padding-block

pr == padding-right

br == border-radius

mb == margin-bottom

g == gap

SCSS Properties 
             $properties: (
                     w: width,
                     h: height,
                     p: padding,
                     px: padding-inline,
                     pr: padding-right,
                     py: padding-block,
                     br: border-radius,
                     mb: margin-bottom,
                     g: gap
             );
Class Filtering System

Control which classes are generated to reduce CSS size:

             $custom-class: true; // Enable filtering
             
             $class-config: (
               w: (50),        // width: only for size 50
               h: (),          // height: no classes generated
               p: (),          // padding: no classes generated
               px: (24),       // padding-inline: only for size 24
               pr: (),         // padding-right: no classes generated
               py: (),         // padding-block: no classes generated
               br: (),         // border-radius: no classes generated
               mb: (),         // margin-bottom: no classes generated
               g: ()           // gap: no classes generated
             );

Empty arrays () = no classes generated for that property

Font Sizing

The font sizing system is based on your Figma design, your UI designer will love you !
By default, a light resize is applied. Adding one or two _ (underscores) will make the resize stronger. 

            $screen: 1440;
            $font-sizes: 12, 16, 24, 36, 52, 72;
.fs-72-px
.fs-72
.fs-72_
.fs-72__
