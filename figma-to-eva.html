Figma File Reading Guide

Understanding how to correctly interpret Figma designs for perfect integration with EVA CSS.

Golden Rule: Exact Correspondence

The number in Figma naming must correspond exactly to the CSS variable. 
// CORRECT
// Figma: g-16 → CSS: var(--16)
gap: var(--16); // corresponds to Figma naming g-16

// INCORRECT  
// Figma: g-16 → CSS: var(--4)
gap: var(--4); // ERROR: doesn't correspond to Figma g-16

Group Identification and Naming

Figma naming examples 
g-16 = Gap of 16px → Use var(--16)
g-32 = Gap of 32px → Use var(--32_)
w-64 = Width of 64px → Use var(--64_)
p-24 = Padding of 24px → Use var(--24)
Size extraction
  • Dimensions: Width/height in Figma inspector
  • Spacing: Group naming (g-16, g-32, etc.)
  • Typography: Font-size of each text element

Concrete Example: Figma to CSS Mapping

Figma → EVA CSS correspondence:
Figma Element Figma Naming/Size EVA CSS Variable Modifier
Main circle (hero) 141px var(--141_) _ (>30)
Palette circle (color-list) 64px var(--64_) _ (>30)
Gap between groups g-32 var(--32_) _ (>30)
Gap label/circles g-16 var(--16) stable (<30)
Gap between circles g-8 var(--8) stable (<30)
Gap circle/label below g-4 var(--4) stable (<30)

Common Errors to Avoid

Misinterpreting naming 
// Figma: g-16 but incorrect usage
gap: var(--4); // Doesn't correspond
Forgetting automatic modifiers 
// Value 32 without adaptive modifier
gap: var(--32); // Should be var(--32_)
Approximate values 
// Figma: 64px but approximation
width: var(--60); // Use exact value var(--64_)

Validation Rules

Exact correspondence

Figma naming = CSS variable

Complete sizes

All Figma values in $sizes

Exact colors

HEX values converted to OKLCH

Zero inline CSS

Everything via variables and classes

Usage Mode Selection

Utility Classes Mode 
$build-class: true;
Ideal for:
  • Rapid development/prototyping
  • Projects with many similar components
  • Teams preferring utility classes (Tailwind style)
  • Need to reuse common styles quickly
Usage: 
<!-- EVAcss generated classes automatically -->
<div class="_bg-brand w-64 p-16">Content with utility classes</div>
<p class="_c-dark fs-16">Text with color class</p>
<button class="_bg-accent_ hover:_bg-accent">Button with opacity</button>
CSS Variables Mode 
$build-class: false;
Ideal for:
  • Very specific/unique design
  • Preference for semantic classes
  • Final CSS optimization (fewer generated classes)
  • Full control over generated CSS
Usage: 
<!-- Semantic classes only -->
<div class="hero-section">Semantic content</div>
<p class="subtitle">Semantic text</p>
<button class="primary-button">Semantic button</button>
// SCSS with EVAcss variables
.hero-section {
  background-color: var(--brand);
  width: var(--64);
  padding: var(--16);
}

2. OKLCH Color Conversion

Hex colors are automatically converted to OKLCH for optimal consistency.

Why OKLCH?

Modern colorimetric format allowing consistent brightness and opacity modifications

Available tool 
scripts/hex-to-oklch.js

Automatic conversion with Claude or manual

Manual command 
npm install culori --save-dev
node scripts/hex-to-oklch.js #ff0000 #ffe500 #7300ff
Conversion result (Ulysse Project): 
oklch(84.1% 0.155 200.4)  // brand (#00d4ff)
oklch(93.6% 0.202 101.8)  // accent (#e5ff00)
oklch(42.8% 0.008 220.3)  // extra (#5f6769)
oklch(95.8% 0.018 84.6)   // light (#f4f5eb)
oklch(15.2% 0.005 220.3)  // dark (#232526)

3. Complete Size Extraction

Analysis of Figma code to identify all dimensions used in the design.

Complete extraction method:
  1. Analyze Figma code → Identify all gap-X, w-[X], h-[X], text-[X], etc.
  2. List sizes → Create complete list of all numeric values
  3. Categorize usage → Identify usage of each size
  4. Visual verification → Compare with design for discrepancies
Example: Analyzed Figma code

  <div className="flex flex-col gap-4">       // 4px gap content
    <p className="text-[120px]">Ulysse2029</p>  // 120px font-size
    <p className="text-[16px]">FIGMA - CLAUDE - EVACSS</p>  // 16px font-size
  </div>
  <div className="flex flex-row gap-32">       // 32px gap colors 
    <div className="size-[141px] rounded-full"></div>  // 141px circles
    <div className="size-[141px] rounded-full"></div>  // 141px circles
  </div>
</div>

Complete SCSS Configuration

Automatic configuration template

Claude automatically generates this configuration based on Figma analysis.

// ===========================================
// EVAcss CONFIGURATION
// ===========================================

// Usage mode selection
$build-class: true;        // true = utility classes / false = CSS variables only
$custom-class: false;      // Reserved for advanced usage
$px-rem-suffix: false;     // false = no debug variables / true = adds -px and -rem variables
$name-by-size: true;       // true = variables named by size (--16) / false = by index (--1)

// ===========================================
// SIZES EXTRACTED FROM FIGMA
// ===========================================

// REQUIRED: All sizes used in the design
$sizes: 4, 8, 16, 32, 64, 120, 141; 

// REQUIRED: All font sizes used
$font-sizes: 16, 120; // 16px (paragraph), 120px (H1 title)

// ===========================================
// UTILITY CLASSES CONFIGURATION
// ===========================================

// Configuration of generated classes (if $build-class: true)
$class-config: (
  w: (),          // Width classes: w-16, w-32, etc.
  h: (),          // Height classes: h-16, h-32, etc.
  p: (),          // Padding classes: p-16, p-32, etc.
  px: (),         // Padding-inline classes: px-16, px-32, etc.
  py: (),         // Padding-block classes: py-16, py-32, etc.
  br: (),         // Border-radius classes: br-16, br-32, etc.
  mb: (),         // Margin-bottom classes: mb-16, mb-32, etc.
  g: (),          // Gap classes: g-16, g-32, etc.
);

// ===========================================
// EVAcss IMPORTS (REQUIRED ORDER)
// ===========================================

@import "../../../styles/framework/eva";      // Core EVAcss + size variables
@import "../../../styles/framework/colors";   // OKLCH color system
@import "../../../styles/framework/theme";    // Theme variables

EVAcss Size Variables

Fluid sizes (automatic responsive)

Base variables 
var(--16)    /* 16px with fluid adaptation */
var(--32)    /* 32px with fluid adaptation */
var(--64)    /* 64px with fluid adaptation */
var(--120)   /* 120px with fluid adaptation */
var(--141)   /* 141px with fluid adaptation */
Size modifiers 
var(--16__)  /* Extreme version (very tight spacing) */
var(--16_)   /* Reduced version (light spacing) */
var(--16)    /* Standard version (normal usage) */
var(--16-)   /* Extended version (generous spacing) */

var(--fs-120)  /* Font-size 120px */
var(--fs-120_) /* Font-size 120px reduced version */
var(--fs-120__) /* Font-size 120px extreme version */

Typography

Responsive modifiers

Classes with modifiers 
<div class="p-16_">Padding 16px reduced version</div>
<div class="p-16-">Padding 16px extended version</div>
<div class="g-32__">Gap 32px extreme version</div>
Responsive behavior
  • __: Maximum adaptation (very tight mobile)
  • _: Strong adaptation (mobile/tablet)
  • none: Standard adaptation
  • -: Minimal adaptation (generous desktop)

HTML integration

Required HTML structure:

    Toggle Dark Mode
  </button>
  
  <!-- Content with thematic classes -->

    Content that automatically adapts to theme
  </div>

JavaScript for persistence

Theme saving and restoration: 
// Function to toggle theme
function toggleTheme() {
  document.body.classList.toggle('toggle-theme');
  const isDark = document.body.classList.contains('toggle-theme');
  localStorage.setItem('theme', isDark ? 'dark' : 'light');
}

// Theme restoration on page load
document.addEventListener('DOMContentLoaded', function() {
  const savedTheme = localStorage.getItem('theme');
  if (savedTheme === 'dark') {
    document.body.classList.add('toggle-theme');
  }
});

// Integration with a button
document.querySelector('.theme-toggle').addEventListener('click', toggleTheme);
